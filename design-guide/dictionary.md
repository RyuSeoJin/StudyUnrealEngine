# UE Archive — 용어 사전 (Dictionary)

> 프로젝트에서 사용하는 고유 명사와 UI 구성 요소를 정의합니다.
> 유저가 작업을 요청할 때 작업 영역이 불분명하면 이 파일에서 먼저 용어를 확인하세요.
> 없는 단어라면 문맥에서 유추 후 유저에게 재확인한 뒤 작업을 진행합니다.

---

## 프로젝트 기본

| 용어 | 정의 |
|---|---|
| **UE Archive** | 이 학습 사이트의 이름. 언리얼 엔진 학습을 위한 단일 HTML 파일 기반 웹 앱 |
| **index.html** | 프로젝트의 유일한 파일. HTML·CSS·JS가 모두 포함된 단일 파일 |
| **서진 / 유저** | 이 사이트의 제작자이자 학습자 |

## 탭 구조

| 용어 | 정의 |
|---|---|
| **상위 탭 (nav-tab)** | 최상단 네비게이션. 대시보드 / 블루프린트 / C++ 세 가지 |
| **하위 탭 (sub-tab)** | 상위 탭 선택 후 나타나는 모드 탭. 개념 정리 / 실습 / 퀴즈 세 가지 |
| **대시보드** | 학습 진행률 통계와 퀵 액션 카드를 보여주는 첫 화면 |
| **블루프린트 탭 (BP)** | 블루프린트 관련 학습 영역. `_subject === 'bp'`일 때 활성 |
| **C++ 탭** | C++ 관련 학습 영역. `_subject === 'cpp'`일 때 활성 |
| **개념 정리** | 개념 카드를 보여주는 모드. `_mode === 'concepts'`일 때 활성 |
| **실습** | 블루프린트/C++ 예제 카드를 보여주는 모드. `_mode === 'practice'`일 때 활성 |
| **퀴즈** | 퀴즈를 진행하는 모드. `_mode === 'quiz'`일 때 활성 |

## 카드 종류

| 용어 | 정의 | 해당 데이터 |
|---|---|---|
| **개념 카드** | 개념 정리 탭에서 보이는 접이식 설명 카드 | `concepts` 배열 |
| **실습 카드 (예제 카드)** | 실습 탭에서 보이는 예제 카드. BP와 C++ 두 종류 | `blueprintExamples` / `cppExamples` 배열 |
| **블루프린트 실습 카드** | 인터랙티브 BP 노드 그래프 + 단계별 안내를 포함하는 실습 카드 | `blueprintExamples` 배열 |
| **C++ 실습 카드** | .h/.cpp 코드 파일을 포함하는 실습 카드 | `cppExamples` 배열 |
| **카드 헤더 (card-head)** | 카드의 클릭 가능한 상단 영역. 클릭 시 카드가 열리거나 닫힘 | `.card-head` |
| **카드 바디 (card-body)** | 카드의 접히는 본문 영역 | `.card-body` |
| **완료 체크** | 카드 오른쪽의 체크박스. 학습 완료 여부를 localStorage에 저장 | `.card-check` |

## 개념 카드 데이터 필드

| 용어 | 정의 |
|---|---|
| **cat** | 개념 카드의 카테고리. `'공통'`/`'블루프린트'`/`'C++'` 중 하나 |
| **공통 개념** | `cat: '공통'` — 블루프린트와 C++ 양쪽 탭에 모두 표시되는 개념 카드 |
| **keywords** | 개념 카드의 검색 키워드 배열. 제목에 없는 핵심 개념어를 한·영으로 담음 (선택 사항) |
| **tagClass** | 카드 태그 배지의 CSS 클래스. `tag-1`(초록/공통) / `tag-2`(파랑/C++) / `tag-4`(앰버/BP) |

## 실습 카드 데이터 필드

| 용어 | 정의 |
|---|---|
| **category** | 실습 카드의 분류. 카테고리 필터 칩에 표시됨 (예: 기초, 입력, 변수) |
| **difficulty** | 난이도. 1=★☆☆ / 2=★★☆ / 3=★★★ |
| **desc** | 예제 설명. "이 예제가 무엇을 하는지" 2–3문장 |
| **hint** | 생각 유도 문구. "먼저 생각해보세요" 박스에 표시됨 |
| **steps** | 블루프린트 예제의 단계별 따라하기 (문자열 배열) |
| **explain** | 예제 해설. 핵심 개념과 응용 방향 설명 |
| **graph** | 블루프린트 예제의 노드 그래프 데이터 (`nodes` + `connections`) |
| **files** | C++ 예제의 코드 파일 배열 (`name`, `type`, `code`) |

## 블루프린트 에디터 (BP 에디터)

| 용어 | 정의 |
|---|---|
| **BP 에디터** | 블루프린트 실습 카드의 인터랙티브 노드 그래프 편집기 (`BlueprintEditor` 클래스) |
| **노드 (Node)** | BP 에디터의 상자 모양 요소. 이벤트/함수/변수/흐름 등 종류가 있음 |
| **핀 (Pin)** | 노드의 연결 지점. 실행 핀(흰 삼각형)과 데이터 핀(색 원) 두 종류 |
| **와이어 (Wire)** | 핀과 핀을 잇는 연결선. 실행=흰색, 데이터=타입별 색상 |
| **실행 핀 (Exec Pin)** | 흰 삼각형(▷) 모양. 실행 흐름을 연결 |
| **데이터 핀** | 색 원 모양. 값(float/string/object 등)을 전달 |

## UI 기능 컴포넌트

| 용어 | 정의 |
|---|---|
| **코드 토글** | 코드 블록 위의 "주석 포함 / 코드만" 전환 탭 (`.code-toggle`) |
| **주석 포함** | 코드 블록의 상세 주석 버전 (기본 표시) |
| **코드만** | 코드 블록의 주석 제거 버전 (기본 접힘) |
| **검색 바** | 개념 정리 탭 상단의 검색 입력 영역 (`#concept-search`) |
| **cat-filter (카테고리 필터)** | 탭 목록 위에서 카테고리를 선택하는 칩 버튼 모음 |
| **정답 보기** | 실습 카드에서 솔루션(그래프/코드/해설)을 펼치는 버튼 |
| **학습 완료** | 실습 카드에서 완료 여부를 기록하는 버튼 |

## 언리얼 엔진 용어 (C++ 개념 관련)

| 용어 | 정의 |
|---|---|
| **리플렉션 (Reflection)** | 런타임에 클래스 구조를 조회할 수 있는 능력. `GENERATED_BODY()`가 제공 |
| **직렬화 (Serialization)** | 객체 상태를 파일에 저장하고 복원하는 것. 레벨 저장에 사용 |
| **GENERATED_BODY()** | 리플렉션·직렬화 코드를 자동 삽입하는 UE 매크로. 클래스 첫 줄에 위치 |
| **접근 지정자** | `public` / `protected` / `private`. 클래스 멤버 접근 권한 제어 |
| **UHT** | Unreal Header Tool. 빌드 전 .h 파일을 훑어 리플렉션 코드를 생성 |
| **UCLASS / UPROPERTY / UFUNCTION** | 클래스·변수·함수를 에디터·블루프린트에 노출하는 UE 매크로 |
| **전방 선언 (Forward Declaration)** | 헤더 포함 없이 클래스 이름만 선언하여 빌드 속도를 높이는 기법 |
| **Super::함수명** | 부모 클래스의 함수 구현을 호출하는 UE 관례. 재정의 함수 첫 줄에 사용 |
