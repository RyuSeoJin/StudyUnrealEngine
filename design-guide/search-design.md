# UE Codex — 검색 기능 가이드 (Search Design)

> 개념 검색 기능의 동작 방식, UI 규칙, 버튼 상태, keywords 배열 규칙을 정리합니다.
> 검색 기능을 수정하거나 확장할 때 이 파일을 읽으세요.

---

## 1. 검색 범위

검색은 **현재 보이는 카드 목록 안에서만** 동작합니다. 탭 전환 시 범위가 자동으로 바뀝니다.

| 현재 탭 | 검색 대상 |
|---|---|
| C++ > 개념 정리 | C++ 카드 + 공통 카드 |
| 블루프린트 > 개념 정리 | 블루프린트 카드 + 공통 카드 |

검색은 **개념 정리 모드**에서만 노출됩니다. 실습·퀴즈 탭에서는 표시하지 않습니다.

---

## 2. 검색 트리거

실시간 필터링(글자 입력마다 실행)을 사용하지 않습니다. 카드가 많아질수록 입력 한 글자마다 처리량이 증가하기 때문입니다.

검색은 아래 두 가지 방법으로만 실행됩니다.

- `[검색]` 버튼 클릭
- 텍스트 필드에서 `Enter` 키 입력

빈 필드에서 Enter를 누르면 검색 초기화 동작이 실행됩니다.

---

## 3. 매칭 방식

검색어가 대상 텍스트에 **포함**되어 있으면 노출합니다 (부분 일치). 대소문자를 무시합니다.

검색 대상 우선순위:
1. 카드 **제목** (title) — 포함 여부 확인
2. 카드 **keywords 배열** — 포함 여부 확인

둘 중 하나라도 매칭되면 카드 노출. 검색 결과 카드는 **접힌 상태**로 표시합니다.

```
"Generated"  검색 → "GENERATED_BODY()" 제목 포함 카드 노출
"리플렉션"   검색 → keywords에 '리플렉션' 있는 카드 노출
"Reflection" 검색 → keywords에 'Reflection' 있는 카드 노출
```

## 4. 버튼 상태와 UI 구조

```
입력 전   [🔍 ________________] [검색 - 회색/비활성]
입력 후   [🔍 리플렉션_________] [검색 - 파란색/활성]
검색 후   [🔍 리플렉션_________] [검색 - 파란색] [검색 초기화 - 붉은색]  결과: 2개
```

| 요소 | 활성 상태 | 비활성 상태 | 노출 조건 |
|---|---|---|---|
| `[검색]` | `--accent-2` (파랑) | `--border` 회색 + disabled | 항상 |
| `[검색 초기화]` | `--coral` (붉은색) | 노출 안 함 | 검색 실행 후에만 |
| 결과 개수 | `--text-dim` (기본) | 노출 안 함 | 검색 실행 후에만. "결과: N개" |

---

## 5. 매칭 하이라이트

검색어와 일치하는 텍스트를 **카드 제목에만** 표시합니다. 본문에는 하이라이트를 적용하지 않습니다.

```css
.search-highlight {
  background: rgba(240,184,96,0.25);
  color: var(--amber);
  border-radius: 2px;
  padding: 0 2px;
}
```

## 6. 검색어 유지 정책

| 상황 | 검색어 처리 |
|---|---|
| 탭 전환 (C++ → 블루프린트 등) | **초기화** — `clearConceptSearch()` 호출 |
| 페이지 새로고침 (F5) | **복원** — `sessionStorage`에서 복원 |
| 브라우저 뒤로가기 후 같은 탭 | **복원** — `sessionStorage`에서 복원 |

`sessionStorage` 키: `ue-codex-search-query`

저장 시점: `executeSearch()` 실행 시.
복원 시점: `applyNav()`에서 개념 정리 뷰를 표시할 때.

## 7. 관련 JS 함수

```javascript
// 상태 변수
let _searchQuery  = '';    // 마지막 실행 검색어
let _searchActive = false; // 검색 결과 표시 중 여부

// 검색 실행 흐름
executeSearch()          // 버튼·Enter → _searchQuery 설정 → renderConcepts()
resetSearch()            // 초기화 버튼·빈필드Enter → 상태 초기화 → renderConcepts()
clearConceptSearch()     // 탭 전환 시 호출 → 상태 + UI 모두 초기화
initConceptSearchEvents()// 검색 바 이벤트 바인딩. _searchBound 플래그로 중복 방지.

// 렌더링 내부
_syncSearchUI(count)     // 버튼·결과수 UI를 상태에 맞게 동기화
_highlightTitle(title, query) // 제목 내 검색어 하이라이트 처리
_escapeRegex(s)          // 검색어의 정규식 특수문자 이스케이프
```

## 8. keywords 배열 작성 규칙

**모든 카드에 추가하지 않습니다.** 제목만으로는 찾기 어려운 개념이 본문에 설명된 카드에만 선택적으로 추가합니다.

```javascript
// 추가 예시 — c-15: 제목은 GENERATED_BODY()지만 본문에 리플렉션·직렬화 설명 있음
{
  id: 'c-15',
  title: "GENERATED_BODY() — 리플렉션과 직렬화",
  keywords: ['GENERATED_BODY()', '리플렉션', '직렬화', 'Reflection', 'Serialization', 'UHT', 'GC'],
  ...
}

// 불필요한 경우 — 제목에 핵심 단어가 이미 있음
{
  id: 'c-07',
  title: "클래스 구조와 접근 지정자",
  // keywords 없음 — 필드 자체를 생략
  ...
}
```

추가 판단 기준:
- 제목에 없는 핵심 개념어가 본문에 등장하는가?
- 한국어 또는 영어 중 한 가지로만 검색해도 찾을 수 있어야 하는가?

두 가지 모두 해당하면 keywords 추가.
